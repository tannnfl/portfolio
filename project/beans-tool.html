Beans and Guns Item Editor

Tool Developer · Visual Tech Art · Unity Editor Extension · C# · 2026
Tool Dev Unity Editor C# ← View Game ← View Design Doc

Overview

Beans and Guns Item Editor is a custom Unity Editor window for authoring and managing all placeable item definitions in the game. With six distinct scenes, each item requires a 3D mesh, icon, prefab, room tag, and spirit dialogue content — data spread across many ScriptableObject assets that the default Inspector cannot efficiently manage at scale across rooms.

The editor aggregates all items into a single panel with per-scene filtering (Esra, Jumie, Katula, Massar, Porda, Yann) and a Sync → Pool feature that pushes defined items into the shared player backpack pool in one click — eliminating manual per-item setup.

Impact

  • Unified item authoring: mesh, icon, prefab, room tag, and dialogue content all editable in one row — no switching between multiple Inspector windows
  • Per-scene filtering lets the content team audit each room's item list independently and catch missing or misassigned room tags instantly
  • Sync → Pool eliminates manual backpack pool maintenance, keeping item definitions in sync with runtime data automatically
  • Preview thumbnails provide visual confirmation, reducing runtime bugs caused by incorrect mesh or icon references

Features

  • Item Definitions — each row displays Item ID, Icon, Prefab, Mesh, Preview thumbnail, Room Tag, and Dialogue Content with inline editing and one-click deletion
  • Room Filter — tab bar filters by six scenes (Esra, Jumie, Katula, Massar, Porda, Yann); All shows the full list
  • Sync → Pool — batch-pushes all item definitions into the shared player backpack pool ScriptableObject
  • + New Item — appends a blank entry to the list, immediately editable
  • Refresh — reloads all ScriptableObject assets to keep the editor in sync with the Project folder
