Document Overview

Design document for Beans and Guns — an online 3D collaborative puzzle sandbox. Covers the full mechanic design for the asynchronous relay system, fog reveal logic, item–scene relationship mapping, and the Unity Editor tooling built to support content authoring.

Contents

  • Asynchronous relay mechanic — per-player item limits, FIFO eviction rules, scene inheritance
  • Fog reveal system — correct object combination conditions and reveal progression
  • Item–scene mapping specification — authoring schema for hint text and object relationships
  • Unity Editor tooling design — hint text editor window, item–scene relationship editor
  • Scene upload/download pipeline — design requirements and handoff to engineering
  • Scene replay system (in development) — timeline playback specification

Design Process

The FIFO eviction rule — oldest items removed when the scene cap is hit — emerged as the most generative constraint in the design. Players can't just accumulate; they must build with intention. Documenting this helped the team understand why the rule was load-bearing and shouldn't be relaxed even when playtesters initially found it frustrating.
