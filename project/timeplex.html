Timeplex

Lead Designer · 2D Grid-Based Puzzle · 2024–2025 · NYU Spring Showcase
Game Design AI Behavior Puzzle Play on itch.io ↗ Design Doc ↗

Overview

  • 2D grid-based puzzle game; screen splits into 1–4 simultaneous timeline frames
  • Players control Erma — trapped in a fractured mental institution — across up to 4 timelines at once
  • All timelines share the same player input; NPC types (Mirror, A*, Guard, Patrol) create divergent outcomes per frame
  • Core mechanic: clicking two timeline frames swaps their order, altering every character's movement sequence in those panels
  • Portals transport characters across timelines; door puzzles require all characters to reach matching exits simultaneously
  • Timeline Continuity: state changes cascade forward — swapping an earlier frame ripples through every later one
  • Narrative delivered through comic-style cutscenes exploring trauma, memory, and healing

Recognition

  • Selected for NYU Spring Showcase

My Role

  • Designed the core mechanic: simultaneous control of 2–4 character instances sharing the same input
  • Authored the mechanics section of the Game Design Document
  • Implemented 4 distinct enemy AI behaviors in Unity: A*, Player Mirror, and two additional types
  • Optimized level design tooling to accelerate iteration
  • Continuously refined the mechanic through playtesting

Design Notes

  • Tension is structural: helping one timeline can trap another — that conflict is the puzzle, not a side effect
  • Frame-swapping adds a meta layer: the player edits the sequence of events that shapes the space, not just navigates it
  • Four NPC types create distinct puzzle archetypes while sharing one unified movement ruleset
  • Mechanic complexity introduced incrementally: movement → doors → object interaction → timeline swap → continuity → portals
