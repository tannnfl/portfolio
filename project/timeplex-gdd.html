Document Overview

Mechanics design document covering the core systems of Timeplex — a 2D grid-based puzzle game built around simultaneous multi-character control. Documents the design rationale, technical specification, and level design guidelines for the mechanics section of the project.

Contents

  • Player character system: four Erma instances across separate timeline windows sharing one input; each holds a single inventory slot
  • 4 NPC behavior specs — Mirror (copies input), Inverse Mirror (opposite direction), Stationary Guard (flashlight sight-line, 3-tile pursuit on detection), Patrol (preset path, sight-line pursuit)
  • Door mechanic: all character instances must occupy matching exits simultaneously; excess characters require incapacitation before passage
  • Portal mechanic: cross-timeline transport with one-way usage counter (departure portal only); blocked when destination is occupied
  • Timeline swap: clicking two frames exchanges their positions, altering each character's surrounding environment and NPC behavior chains
  • Timeline continuity: state changes cascade forward — reordering an earlier frame retroactively applies effects to all later frames
  • Mechanic progression sequence: Movement → Doors → Object Interaction → Connected Movement → Level Exit → Timeline Swap → Timeline Continuity → Portals

Design Process

The mechanic was designed around a single insight: actions that help one character instance can trap another. The GDD documents how we found configurations where this tension is the puzzle — not a frustration. The enemy AI specs were written alongside the Unity implementation, bridging design intent with technical constraints.
